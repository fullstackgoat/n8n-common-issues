00  /  Why this exists

Show, don't tell.

I'm applying for the Senior Support Engineer role at n8n. Rather than send a resume that says "I can debug things," I'm shipping artifacts that show I've already been doing the job — for free, in public, against the live forum. For each of the 10 ticket clusters in top-10-issues.html, I commit to producing a reproduction, a knowledge-base article, and a public answer on the forum. That's 30 deliverables. The 12 projects below are the containers for those deliverables, organized to maximize signal per hour.

Reading the cards

Every project has: a one-paragraph description, a concrete deliverable, the JD bullets it proves, the issue numbers it covers, and a realistic time-box.

The recommended order

Start with Project 3 (forum push — do it today), then Project 1 (repro lab — the centerpiece), then Project 2 (KB articles). The other nine layer on top.

Highest-ROI single move

Project 3. Spending 45 min/day answering open forum threads and landing on the Support Leaderboard is the most direct, audit-able evidence of the role you can produce.

Project 01
repro
lab
5 days

The centerpiece — n8n-repro-lab

Public monorepo · One folder per ticket cluster

A public GitHub monorepo where each subfolder is a minimum reproduction of a real ticket from the forum. Each repro ships:

  • docker-compose.yml that boots a clean stack (n8n + dependencies)
  • REPRO.md — exact steps, observed vs. expected output
  • ROOT_CAUSE.md — why it happens, with citations to source code or release notes
  • WORKAROUND.md — what to tell the customer today
  • FIX.md when a patch exists upstream — link to the PR or commit
Concrete deliverable Repo at github.com/<you>/n8n-repro-lab with at least 10 working repros, one per Top-10 cluster.

"Strong troubleshooting and problem-solving skills"

Project 04
debug
cli
3 days

Tooling that scales me — n8n-debug-cli

Node.js · npm package · ~300 LOC

A small CLI that turns repeated support work into one-line commands:

n8n-debug repro <cluster>          # boots a docker-compose repro from the lab
n8n-debug logs  <execution.json>   # parses an execution export, surfaces error/stack/timing outliers
n8n-debug webhook <url> [--secret] # sends test payloads with HMAC, header inspection
n8n-debug oauth <url>              # decodes JWT, traces redirect chain, checks callback URL
n8n-debug net   <host>             # one-shot DNS/TLS/HTTP/proxy diagnostic
Concrete deliverable Repo at github.com/<you>/n8n-debug-cli with README, npm package, GitHub Actions CI.

JD bullets it proves

"Experience working with JavaScript, Node.js, or similar technologies"

05  /  What this portfolio is not

Deliberate omissions

It is not

— A flashy product or AI demo (wrong role)
— A long architecture treatise (you're applying to a support team, not a platform team)
— A massive open-source project that takes 6 months (you need to apply this month)

It is, intentionally

— Reproducible work that mirrors the daily job
— Documentation that helps real users today
— Forum reputation the hiring panel can audit live during the interview
